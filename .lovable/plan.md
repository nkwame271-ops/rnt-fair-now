

# Fix: Settlement Account Save Button Not Working

## Problem
The "Save" button on each System Settlement Account card is missing its `onClick` handler. The `handleSaveSettlement` function exists and works correctly, but it's simply never wired to the button. This is why changes appear unchangeable — you can edit the fields, but clicking Save does nothing.

The Paystack Recipient Code field is optional and auto-generated by the system when the first payout is triggered. You do not need to fill it in manually.

## Fix

**File: `src/pages/regulator/OfficePayoutSettings.tsx`** (line 359)

Add the missing `onClick` handler to the settlement account Save button:

```tsx
<Button
  onClick={() => handleSaveSettlement(key)}   // ← THIS IS MISSING
  disabled={savingSettlement === key}
  variant={hasEdits ? "default" : "outline"}
  className="w-full sm:w-auto"
>
```

That's it — one line fix. The save logic (`handleSaveSettlement`) already handles insert vs update correctly.
